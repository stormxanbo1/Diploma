Ниже даю подробное текстовое описание сценариев работы (Use Cases) для каждой роли — что именно и где может делать пользователь в приложении. Это поможет фронтендеру проработать все экраны и действия в Figma.

---

## ADMIN

1. **Вход в систему**
   — Заходит на страницу логина, вводит email/пароль → перенаправление на Admin Dashboard.

2. **Admin Dashboard**
   — Видит четыре карточки-счётчика: общее число пользователей, проектов, задач и непрочитанных уведомлений.
   — Кликает на карточку «Users» → попадает в раздел управления пользователями.
   — Кликает на «Projects», «Tasks», «Groups», «Resources», «Reports», «Notifications» — переход к соответствующим разделам.

3. **Управление пользователями (Users Management)**
   — Видит таблицу всех пользователей.
   — Нажимает «Create User» → открывается форма с полями Username, Email, Password, выбор ролей, выбор группы и отдела. Заполняет и сохраняет → новый пользователь появляется в списке.
   — В списке нажимает «Edit» напротив строки пользователя → открывается та же форма, редактирует email/роль/группу → сохраняет.
   — Нажимает «Delete» → подтверждает удаление → пользователь исчезает из списка.

4. **Управление группами (Groups Management)**
   — Видит список групп.
   — «Create Group» → вводит название, выбирает пользователей из multiselect → сохраняет.
   — «Edit» открывает форму с текущим названием и списком участников → меняет состав → сохраняет.
   — «Delete» удаляет группу (после подтверждения).

5. **Управление ресурсами (Resources Management)**
   — Видит список всех ресурсов (каталоги, формы).
   — «Create Resource» → вводит имя, тип (каталог/форма), URL, указывает в multiselect, какие роли и группы могут получить доступ → сохраняет.
   — «Edit»/«Delete» аналогично.

6. **Проекты и задачи**
   — Заходит в раздел Projects или Tasks (структура как у STAFF, но при этом у ADMIN всегда доступны кнопки «Edit» и «Delete» для любых записей).
   — Может удалять/редактировать чужие проекты и задачи.

7. **Управление расписанием (Schedule Management)**
   — Открывает раздел расписаний: видит таблицу всех ScheduleEntry.
   — «Create» → выбирает группу, преподавателя, предмет, дату/время и локацию → сохраняет.
   — «Edit»/«Delete» аналогично удаляет или изменяет расписание.

8. **Отчёты (Reports)**
   — Переходит в Reports, задаёт фильтр (дата от/до, группа).
   — Нажимает «Generate Tasks Report» → видит таблицу и/или график с количеством задач, средним временем выполнения и суммарным весом.
   — Аналогично «Generate Activity Report» — статистика по действиям пользователей.

9. **Уведомления (Notifications)**
   — Открывает Notifications: видит список уведомлений всех типов, где можно поставить галочки напротив нескольких и кликнуть «Mark Selected as Read» или «Mark All as Read».
   — По клику на уведомление может перейти «View Details» → перейти к связанной задаче или расписанию.

## STAFF

1. **Login → Staff Dashboard**
   — Видит блоки: «Мои проекты», «Мои задачи», «Предстоящие дедлайны».

2. **Проекты (Projects)**
   — На странице Projects показывает только те проекты, где он указан как owner или участник.
   — Вверху — фильтры по категории и кнопка «Create Project».
   — «Create Project» → форма Title, Description, Deadline (дата-пикер), Category (select), Participants (multiselect пользователей) → «Save».
   — В списке у своих проектов — кнопки «Edit» и «Delete».
   — Клик по строке — переход на страницу просмотра проекта с кнопкой «Edit» (если он owner) и списком связанных задач.

3. **Задачи (Tasks)**
   — На Tasks видит все задачи, в которых участвует. Фильтр по проекту и по исполнителю (себе).
   — Кнопка «Create Task» открывает модальное окно с полями: Title, Description, Deadline, Priority, Weight, Project (select, необязательно), Assignees (multiselect) → «Create».
   — В списке у своих задач есть «Edit», «Delete», «View Comments», «View Attachments».
   — «Edit» открывает ту же форму, где меняются статус, приоритет, вес, дедлайн и список assignees.
   — «Comments»: при клике открывается подсписок комментариев, поле «Add Comment» + кнопка «Post».
   — «Attachments»: кнопка «Upload File» → загрузка в Minio, после — список вложений с «Download» и «Delete».

4. **Уведомления**
   — Видит на Dashboard количество новых уведомлений.
   — Открывает раздел Notifications: список с не­прочитанными сначала, кнопка «Mark All as Read».

5. **Отчёты**
   — Переходит в Reports: фильтры «from», «to» (по датам).
   — «Generate Tasks Report» — видит статистику только по своим задачам (кол-во, время, вес).
   — «Generate Activity Report» — статистика действий за период.

## TEACHER

1. **Login → Teacher Dashboard**
   — Виджеты: «Расписание на сегодня» (сегодняшние ScheduleEntry для групп, которые ведёт преподаватель), «Загрузить материал».

2. **Просмотр расписания (Schedule Viewer)**
   — Открывает Schedule: таблица с фильтрами по группе и дате.
   — Может сменить дату и группу, чтобы увидеть расписание на другой день/для другой группы.

3. **Материалы (Resources)**
   — Открывает Resources: видит только те ресурсы и вложения, которые ему разрешены и которые он сам загружал.
   — Кнопка «Upload Material» рядом с названием ресурса → выбирает файл и «Upload».
   — В списке вложений у своих материалов доступны «Download» и «Delete».
   — Не видит кнопки «Create Resource» (только ADMIN).

4. **Задачи и комментарии**
   — При назначении учителем в проекте/задаче он попадает в списки Tasks аналогично STAFF.
   — Может просматривать, редактировать статус и оставлять комментарии.

5. **Уведомления**
   — Как у STAFF: всплывающее число на Dashboard, раздел Notifications, «Mark All as Read».

## STUDENT

1. **Login → Student Dashboard**
   — Виджеты: «Расписание на сегодня» (ScheduleEntry для группы студента), «Мои задачи», «Доступные ресурсы».

2. **Просмотр расписания**
   — На Schedule Viewer по умолчанию открывается расписание его группы на текущую дату.
   — Можно выбрать другую дату, чтобы увидеть расписание на любой день.

3. **Ресурсы и формы**
   — Открывает Resources: видит все ресурсы (каталоги, формы), доступные его роли и группе.
   — У каждого ресурса кнопка «Download».
   — Не имеет прав на загрузку или удаление.

4. **Задачи (Tasks)**
   — Видит только задачи, в которых он значится как assignee.
   — Могут быть задачи с projectId и без.
   — В списке: Title, Deadline, Status, Priority.
   — Может кликнуть «Mark as Done» (если статус не DONE) или «View Details».
   — View Details: показывает полное описание, статус, приоритет, вес, дедлайн, список комментариев и вложения.
   — В поле комментариев может оставить свой комментарий (если нужно уточнить что-то).
   — Не видит кнопок «Create», «Edit» или «Delete».

5. **Уведомления**
   — Видит счётчик новых уведомлений на Dashboard.
   — Открывает Notifications: список, кнопка «Mark All as Read».

---

> **Примечание для всех ролей:**
> — При попытке перейти в раздел или выполнить действие без прав (например, STUDENT пытается создать задачу) показывать ошибку 403 Forbidden и информировать пользователя (баннером или модальным окном) о недостатке прав.
> — Все формы должны валидировать обязательные поля (e.g. Title, Deadline) и показывать ошибки inline.
> — Любая успешная операция (создание/редактирование/удаление) подтверждается уведомлением (toastr/snackbar) о том, что действие выполнено.
